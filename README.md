Instagram Uploader Bot
This project automates the process of uploading videos from a Google Drive folder to an Instagram account. It is written in Python and is designed to be deployed as a Docker container, making it suitable for continuous cloud-based operation. Upload history is persisted remotely via a Supabase database, which eliminates the need for a local SQLite file and allows the bot to run statelessly across container restarts.

How It Works
When the script runs, it begins by decoding the Google credentials from environment variables and writing them to disk, then authenticates with the Google Drive API using OAuth 2.0. It retrieves the list of video files present in the configured Drive folder and iterates through each one. For every video, it queries Supabase to determine whether that file has already been uploaded. If it has, the video is skipped. If it is new, the script downloads the file locally, logs into Instagram — reusing a saved session where possible — and uploads the video as a Reel with a caption derived from the filename. A record of the upload is then written to Supabase, and the local file is deleted. To reduce the risk of triggering Instagram's rate limits, the bot enforces a minimum one-minute gap between consecutive uploads.

Project Structure
The codebase is divided across several modules. main.py serves as the entry point and orchestrates the full pipeline. google_drive.py handles OAuth authentication with Google and exposes functions for listing and downloading video files. instagram.py manages Instagram login and video upload via the instagrapi library, with session persistence handled through a local sessions.json file. database.py encapsulates all interactions with Supabase, including upload record insertion, duplicate checking, timestamp retrieval, and rate-limit enforcement. The Dockerfile defines the container environment, including system-level dependencies such as ffmpeg which is required for video processing.

Dependencies
The project requires the following Python packages, all listed in requirements.txt: python-dotenv for environment variable management, supabase for database connectivity, google-api-python-client, google-auth, and google-auth-oauthlib for Google Drive integration, instagrapi for Instagram automation, and rich for enhanced terminal output.

Configuration
The application is configured entirely through environment variables, which should be set in the deployment environment or in a local .env file. The required variables are as follows.
GOOGLE_DRIVE_FOLDER_ID must be set to the ID of the Google Drive folder containing the videos to be uploaded. This ID is the final path segment of the folder's URL. INSTAGRAM_USERNAME and INSTAGRAM_PASSWORD should contain the credentials for the target Instagram account. SUPABASE_URL and SUPABASE_KEY must point to the Supabase project used for upload tracking. Finally, RAILWAY_GOOGLE_CREDENTIALS and RAILWAY_GOOGLE_TOKEN should contain the base64-encoded contents of credentials.json and token.json respectively, allowing the container to reconstruct these files at runtime without baking sensitive data into the image.

Supabase Setup
The Supabase project must contain a table named uploaded_videos with at minimum two columns: file_id of type text, which stores the Google Drive file ID, and timestamp_utc of type text, which records the time of upload in ISO 8601 format. No local database initialisation is required — the remote table must simply exist and be accessible via the provided credentials.

Running with Docker
The recommended deployment method is Docker. The image is built from the repository root and configured to run main.py on container start. All secrets are injected as environment variables at runtime rather than being embedded in the image. Because Supabase is used as the backing store, no volume mounts are required to persist state between runs — the container is effectively stateless.

Running Locally
For local testing, install the dependencies with pip install -r requirements.txt, populate a .env file with the variables described above, ensure that valid credentials.json and token.json files are present in the project directory, and execute python main.py. On the first run against a fresh Google authorisation, the OAuth flow will need to be completed interactively in the browser.
